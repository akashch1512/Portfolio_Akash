# Akash Chaudhari Portfolio

Clean, single-page portfolio built with Flask. Includes a footer-only second section, a legal docs modal, and an email notification signup stored in Supabase.

**Features**
- Single-page hero with scroll hint and a compact footer section
- Legal docs modal with dynamic sections (edit in `static/cursor.js`)
- Notification signup with AJAX feedback (no page reload)
- Responsive layout for desktop and mobile

**Tech Stack**
- Flask + Jinja templates
- HTML/CSS/JS
- Supabase (email notifications)

**Local Setup**
1. `python -m venv .venv`
1. `.\.venv\Scripts\Activate.ps1`
1. `pip install -r requirements.txt`
1. Set environment variables (see below)
1. `python app.py`
1. Open `http://localhost:8000`

**Environment Variables**
- `SUPABASE_URL` required for notification signup
- `SUPABASE_KEY` required for notification signup
- `PORT` optional, default `8000`
- `FLASK_DEBUG` set to `1` for debug mode

**Supabase Table**
- Table name: `notification_user`
- Column: `mail_id` (text)

Example SQL:
```sql
create table if not exists notification_user (
  id bigint generated by default as identity primary key,
  mail_id text not null,
  created_at timestamp with time zone default now()
);
```

**Docker**
1. `docker build -t portfolio .`
1. `docker run -p 8000:8000 --env-file .env portfolio`

**Customize**
- Hero content and footer layout: `templates/index.html`
- Styling: `static/styles.css`
- Legal docs content: `static/cursor.js` (edit `LEGAL_CONTENT`)

**Project Structure**
- `app.py` Flask entrypoint
- `templates/index.html` single-page UI
- `static/styles.css` styling
- `static/cursor.js` animations + modal + form logic
- `database/schema.py` Supabase insert helper
